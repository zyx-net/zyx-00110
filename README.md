# 设备巡检停复机登记系统

## 项目概述

本系统是一个基于Python Tkinter的设备巡检停复机登记桌面工具，用于管理设备的异常报告、停机审批、维修登记和复机审批流程。

## 功能特性

- 设备管理：添加、删除设备，查看设备状态
- 状态流转：正常 → 待停机审批 → 已停机 → 维修中 → 待复机审批 → 已复机 → 复发观察
- 维修记录：记录维修内容、维修人员和维修时间
- 审批记录：记录停机/复机审批意见和审批人
- 阈值配置：配置监控阈值
- 数据导出：导出所有记录到JSON文件
- 数据持久化：所有数据本地存储，关闭再打开后数据不丢失

## 状态流转图

```
正常 ──(报告异常)──→ 待停机审批 ──(申请停机)──→ 已停机
   ↑                                                   │
   │                                                   ↓
   └──(恢复正常)── 复发观察 ←──(复发观察)── 已复机 ←──(批准复机)── 待复机审批
                                                          ↑
                                                          │
                                                   (完成维修)── 维修中
```

## 实操步骤

### 主流程验证

#### 1. 添加设备
- 在"设备管理"标签页中，输入设备ID（如 `DEV001`）和设备名称（如 `水泵一号`）
- 点击"添加设备"按钮

#### 2. 巡检发现异常
- 选中刚添加的设备
- 点击"报告异常"按钮
- 在弹出对话框中输入异常描述（如 `水泵异响，振动超标`）
- 点击确定，设备状态变为"待停机审批"

#### 3. 申请停机
- 选中"待停机审批"状态的设备
- 点击"申请停机"按钮
- 在弹出对话框中输入停机原因（如 `需检修轴承`）
- 点击确定，设备状态变为"已停机"

#### 4. 开始维修
- 选中"已停机"状态的设备
- 点击"开始维修"按钮
- 设备状态变为"维修中"

#### 5. 登记维修记录
- 选中"维修中"状态的设备
- 点击"完成维修"按钮
- 在弹出对话框中输入维修内容（如 `更换轴承，润滑处理`）
- 输入维修人员（如 `张三`）
- 点击确定，设备状态变为"待复机审批"

#### 6. 主管批准复机
- 选中"待复机审批"状态的设备
- 点击"批准复机"按钮
- 在弹出对话框中输入审批意见（如 `维修完成，同意复机`）
- 输入审批人（如 `李四`）
- 点击确定，设备状态变为"已复机"

#### 7. 导出记录
- 切换到"系统配置"标签页
- 点击"浏览"按钮选择导出目录（或使用默认目录）
- 点击"导出记录"按钮
- 记录将导出为JSON文件

### 失败链路验证

#### 1. 没有维修记录却申请复机
- 添加一个新设备
- 直接将设备状态改为"待复机审批"（通过数据文件手动修改）
- 选中该设备，点击"批准复机"
- 系统提示："没有维修记录，不能批准复机"

#### 2. 已停机设备再次重复停机
- 选中"已停机"状态的设备
- 点击"申请停机"按钮
- 系统提示："设备已停机，不能重复停机"

#### 3. 导出目录改成非法路径
- 切换到"系统配置"标签页
- 在"导出目录"输入框中输入不存在的路径（如 `C:\nonexistent\dir`）
- 点击"保存配置"按钮
- 系统提示："导出目录不存在"，原有设置不会被冲掉

## 数据持久化

所有数据存储在 `data/` 目录下的JSON文件中：

- `devices.json`：设备信息
- `repair_records.json`：维修记录
- `approval_records.json`：审批记录
- `config.json`：系统配置（导出目录、阈值等）

关闭应用后再打开，所有数据将自动加载恢复。

## 注意事项

1. 请确保运行目录有读写权限
2. 导出目录必须是已存在的有效目录
3. 设备ID不能重复
4. 状态流转必须按照规定的顺序进行
